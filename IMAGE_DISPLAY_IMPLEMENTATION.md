## What's Been Implemented

### 1. Reusable Image Widget
**File:** `lib/widgets/network_image_with_fallback.dart`

A flexible widget that handles:
- Network image loading with progress indicator
- Error states with icon fallback
- Custom fallback placeholder support
- Rounded corners support
- Null/empty URL handling

**Usage:**
```dart
NetworkImageWithFallback(
  imageUrl: coverImageUrl,
  height: 200,
  borderRadius: BorderRadius.circular(12),
  fallbackIcon: CupertinoIcons.photo,
)
```

### 2. Tour Card with Image Support
**File:** `lib/design_system/components/pg_card.dart`

- Added `coverImageUrl` parameter to `PGTourCard`
- Displays cover image when available
- Falls back to map icon when no image provided
- Handles image load errors gracefully

**Usage:**
```dart
PGTourCard(
  title: tour.title,
  subtitle: tour.city,
  duration: '3 days',
  poiCount: 12,
  coverImageUrl: tour.images?.cover?.url, // When API is ready
  onTap: () {},
)
```

### 3. Tour Detail Page Header
**File:** `lib/main.dart` (TourWithTranscriptScreen)

- Cover image section prepared (currently commented out)
- Will display at top of tour detail page when API is ready
- Falls back gracefully when no image available

## Image URL Format

Images from the API will have URLs like:
- Tour cover: `/tours/{tour_id}/images/cover.jpg`
- POI cover: `/pois/{city}/{poi_id}/images/image_001.jpg`

These need to be prefixed with the API base URL:
```dart
final fullUrl = '${ApiService.baseUrl}${relativeUrl}';
```

## Fallback Behavior

### When No Image Available:
- **Tour Cards**: Show map icon in gray box
- **Tour Detail**: No image section shown
- **POI Items**: No image shown (graceful degradation)

### When Image Load Fails:
- Show photo icon placeholder
- No error thrown - graceful degradation
